# Campus Traffic Intelligent Distribution

## Опис проєкту

Цей репозиторій містить програмну реалізацію практичної частини курсової роботи на тему:

**«Інтелектуальний розподіл трафіку між точками доступу Wi-Fi і Ethernet-сегментами»**

Проєкт реалізує Python-модель кампусної мережі, у якій досліджується розподіл різних типів трафіку між Wi-Fi точками доступу та Ethernet-сегментами. Модель дозволяє порівняти кілька алгоритмів розподілу потоків і оцінити їх ефективність за показниками затримки, завантаження мережі, втрат пакетів, QoS-задоволення та кількості перевантажених сегментів.

## Мета проєкту

Метою програмної реалізації є моделювання роботи змішаної Wi-Fi/Ethernet кампусної мережі та дослідження ефективності інтелектуального алгоритму розподілу трафіку.

Програма дозволяє:

* змоделювати кампусну мережу з Wi-Fi точками доступу та Ethernet-сегментами;
* створити потоки різних типів трафіку;
* порівняти кілька алгоритмів розподілу;
* оцінити середню затримку, втрати пакетів і завантаження сегментів;
* визначити кількість перевантажених сегментів;
* оцінити частку потоків, обслужених відповідно до вимог QoS;
* побудувати графіки для аналізу результатів.

## Основна ідея

У змішаній кампусній мережі частина користувачів працює через Wi-Fi точки доступу, а частина — через Ethernet-сегменти. Різні типи трафіку мають різні вимоги: відео та голосовий зв’язок чутливі до затримки й втрат, навчальні платформи потребують стабільного доступу, а фонові оновлення можуть мати нижчий пріоритет.

Запропонована модель дозволяє дослідити, як різні алгоритми розподіляють потоки між доступними сегментами мережі. Основний інтелектуальний алгоритм використовує багатокритеріальну оцінку сегмента та враховує пропускну здатність, поточне завантаження, затримку, втрати пакетів, пріоритет трафіку та ризик перевантаження Wi-Fi.

## Реалізовані можливості

У проєкті реалізовано:

* графову модель кампусної Wi-Fi/Ethernet мережі;
* Wi-Fi точки доступу з різною пропускною здатністю;
* Ethernet-сегменти з різними характеристиками;
* генерацію потоків різних типів трафіку;
* моделювання кількох сценаріїв навантаження;
* порівняння алгоритмів розподілу трафіку;
* розрахунок основних метрик ефективності;
* збереження результатів у CSV-файли;
* побудову графіків для курсової роботи.

## Типи трафіку

У моделі використовуються такі типи трафіку:

| Тип трафіку       | Опис                            |
| ----------------- | ------------------------------- |
| voice             | голосовий трафік                |
| video             | відеолекції та відеоконференції |
| lms               | навчальна платформа             |
| web               | звичайний веб-трафік            |
| file_download     | завантаження файлів             |
| background_update | фонові оновлення                |

Для кожного типу трафіку задаються пріоритет, потрібна пропускна здатність, допустима затримка та допустимий рівень втрат.

## Алгоритми, що порівнюються

У програмній моделі реалізовано чотири підходи до розподілу трафіку.

### Baseline

Базовий алгоритм, який спрямовує потоки через локальний сегмент без інтелектуальної оптимізації. Він використовується як контрольний варіант для порівняння.

### Least Loaded

Алгоритм вибирає сегмент із найменшим поточним завантаженням. Такий підхід дозволяє зменшити ризик перевантаження, але не повністю враховує вимоги різних типів трафіку.

### QoS Priority

Алгоритм враховує пріоритет трафіку, допустиму затримку та втрати пакетів. Він краще підходить для обслуговування критичних потоків, таких як голосовий і відеотрафік.

### Intelligent

Основний алгоритм роботи. Він використовує багатокритеріальну оцінку доступних сегментів і враховує:

* доступну пропускну здатність;
* поточне завантаження сегмента;
* очікувану затримку;
* очікувані втрати пакетів;
* пріоритет трафіку;
* штраф за перевантаження Wi-Fi.

Потік спрямовується в сегмент із найкращою інтегральною оцінкою.

## Сценарії дослідження

У роботі використано п’ять сценаріїв:

1. **normal_load** — нормальне навантаження;
2. **peak_load** — пікове навантаження;
3. **video_dominant** — домінування відеотрафіку;
4. **wifi_overload** — перевантаження Wi-Fi точок доступу;
5. **qos_stress** — стресовий сценарій для перевірки QoS.

## Метрики оцінювання

Для аналізу результатів використовуються такі показники:

* середня затримка;
* середні втрати пакетів;
* середнє завантаження Wi-Fi точок доступу;
* середнє завантаження Ethernet-сегментів;
* максимальне завантаження сегмента;
* кількість перевантажених сегментів;
* частка потоків, обслужених відповідно до вимог QoS;
* інтегральний індекс ефективності.

## Опис основних модулів

### main.py

Головний файл запуску проєкту. Запускає експерименти, зберігає результати та будує графіки.

### config.py

Містить основні параметри моделі: типи трафіку, сценарії, ваги критеріїв для інтелектуального алгоритму та пороги перевантаження.

### models.py

Описує основні сутності системи: мережеві сегменти, потоки трафіку, призначення потоків і результати симуляції.

### topology.py

Створює модель кампусної мережі та графову схему Wi-Fi/Ethernet інфраструктури.

### traffic_generator.py

Генерує потоки трафіку для різних сценаріїв навантаження.

### algorithms.py

Містить реалізацію алгоритмів розподілу трафіку: baseline, least_loaded, qos_priority та intelligent.

### experiment.py

Запускає експериментальні сценарії та порівнює алгоритми між собою.

### metrics.py

Розраховує показники ефективності роботи алгоритмів.

### visualization.py

Будує графіки та схему топології мережі.

## Встановлення залежностей

Рекомендовано використовувати віртуальне середовище Python.

```bash
python -m venv .venv
```

```bash
.\.venv\Scripts\Activate.ps1
```

Встановлення бібліотек:

```bash
pip install -r requirements.txt
```

## Запуск проєкту

Для запуску моделі потрібно виконати:

```bash
python main.py
```

Після запуску програма створить або оновить файли в папці `results/`.

## Результати

У папці `results/` зберігаються:

* зведена таблиця результатів;
* детальна таблиця розподілу потоків;
* графік порівняння середньої затримки;
* графік QoS-задоволення;
* графік кількості перевантажених сегментів;
* графік інтегрального індексу ефективності;
* графіки завантаження Wi-Fi та Ethernet;
* схема змодельованої топології мережі.

Основний результат дослідження полягає в тому, що інтелектуальний алгоритм забезпечує меншу середню затримку, нижче завантаження Wi-Fi точок доступу, відсутність перевантажених сегментів і вищий рівень QoS-задоволення порівняно з базовим підходом.

У сценарії перевантаження Wi-Fi базовий алгоритм показує значне зростання затримки та появу перевантажених сегментів. Інтелектуальний алгоритм перерозподіляє частину потоків на доступні Ethernet-сегменти та менш завантажені канали, завдяки чому підвищується стабільність роботи мережі.

Такий підхід може бути корисним для аналізу кампусних мереж, навчальних закладів, офісних мереж і локальних інфраструктур із різними типами трафіку.
